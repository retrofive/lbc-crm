01 Overview

LBC Collection Manager is a single-file portable web application designed for LBC Cargo Branch 602 agents in Florida. It runs entirely in any modern browser (Chrome or Edge recommended) directly from a local HTML file — no server, no internet connection required for day-to-day use.

All data is stored in a JSON file on your computer that you choose and control. Multiple computers can share the same JSON file over a network drive or synced folder, allowing a team to work from a common dataset.

FeatureDescription
Active CollectionRecord box entries in real time on pickup day
HistoryView, audit, and export all completed collections
ReportsSortable summaries by month, box size, checks, and detail line items
AnalyticsPeriod-over-period comparisons and trend charts
SearchFull-text search across all BOL numbers, customers, and check numbers
ContactsCustomer address book linked to shipment history
Boxes InventoryTrack individual box status and outgoing log
CalculatorPop-out 10-key calculator with keyboard support
ODS ImportBulk import from legacy spreadsheet files with pricing override
02 Getting Started
2.1   First Launch

Open LBC-CRM.html in Chrome or Edge by double-clicking the file. The app loads instantly. On the very first launch, data lives only in the browser's local storage — it is not yet connected to a persistent file on disk.

⚠ Local storage can be cleared by the browser. Always connect a data file (see below) to ensure your data is safely saved to disk.
2.2   Creating a Data File

To save your data persistently to a file on disk:

  • Open Settings (⚙ gear icon, top-right corner).
  • Under Data, click Create New Data File.
  • A Save As dialog appears. Choose a folder and name the file (e.g. LBC-CRM-data.json). Click Save.
  • The footer turns green and shows the filename — you are now connected.
  • From this point, every save automatically writes to that file.
💡 Store the JSON file in a shared network folder or cloud-synced directory (OneDrive, Google Drive) to enable multi-PC access.
2.3   Connecting to an Existing Data File

On a second computer, or after moving the file, connect the app to the existing data:

  • Open Settings → Connect to Existing File.
  • An Open dialog appears. Browse to and select your .json data file.
  • The app loads all data from the file. The footer shows the filename in green.
⚠ This is an Open dialog, not Save As — it never overwrites your file on connect. Data is read first, then the connection is established.
2.4   Auto-Reconnect on Reload

The app remembers your last-used data file using the browser's IndexedDB storage. When you reload or reopen the HTML file:

  • If the browser still holds permission (same session): reconnects silently, no click needed.
  • If permission expired (new browser session): a blue banner appears at the top with a single Reconnect button. One click re-grants permission and loads data.
  • If the file was deleted or moved to another drive: the banner appears, but clicking Reconnect will show "File no longer exists — please connect a new data file." The stored handle is cleared and the footer turns grey.
💡 Moving the file to a different folder on the same drive is fine — the browser handle follows it automatically.

3.2   Calculator

A pop-out 10-key calculator is accessible via the calculator icon in the header. It supports both mouse clicks and full keyboard input:

  • Number row (0–9) and numeric keypad
  • Operators: + - * / (keyboard or on-screen)
  • Enter or NumpadEnter to calculate
  • Backspace to delete last digit, Escape to clear
  • Tape history scrolls within the popup — the calculator can be dragged to any position on screen
3.3   Refresh Button (↺)

The circular arrow button next to the calculator manually re-reads the connected JSON file from disk and reloads all data. Use this to pull in changes made by another computer on a shared file.

The app also auto-refreshes whenever you switch back to the browser tab after having it in the background.

💡 If no file is connected, the button shows a toast: "No data file connected."
3.4   Settings (⚙ Gear)

The gear icon opens the Settings & Tools panel, organized into sections: Account (Branch Settings, Pricing), Data (file management, backup, ODS import), and App (update checker). Close by clicking × or clicking outside the panel. Each section is covered in detail in Section 11.

3.5   Footer Status Bar

The footer at the bottom of every page shows your data file connection status at a glance:

  • ● Green dot + filename: a data file is connected and active (e.g. LBC-CRM-data.json).
  • ● Grey dot + "NO FILE CONNECTED": no data file is linked. Data is only in browser local storage and at risk of loss on a browser clear.
⚠ The browser API only exposes the filename, not the full folder path. Name your file descriptively (e.g. LBC-602-main.json) to avoid confusion when multiple files exist.
04 Active Collection

The Active tab is your live workspace on pickup day. One collection is open at a time. Boxes are added one by one as customers arrive.

4.1   Starting a Collection

When no collection is open, the Active tab prompts you to start one. Enter the collection date, branch number, and agent name, then click Start Collection. The entry form appears immediately.

4.2   Adding Box Entries

Fill in the entry row at the top of the table:

FieldDescription
BOL NumberBill of Lading number — must be 8 or more digits
CustomerLast name, First name (or full name)
DestinationCity or region (optional)
SizeLarge / Half / Jumbo / Jr Box / Irregular (Irr1–Irr7)
CostAuto-fills from the pricing table when Size is selected
NetOverride the cost if a discount applies
PaymentCash, Check, or Split (part cash / part check)
Check #Appears when Payment = Check or Split; up to 5 digits

Press Enter or click Add to save the row. The cursor returns to the BOL field for the next entry.

4.3   Payment Types
Cash

Select Cash from the Payment dropdown. No check number required. The cost is set by the pricing table.

Check

Select Check and enter the check number (up to 5 digits). Check numbers are always 5 digits or fewer; BOL numbers are always 8 or more — these two ranges never overlap.

Split

Select Split… to open the split payment dialog. Enter the cash amount, check amount, and check number. A mismatch warning appears if the total does not equal the box cost.

4.4   Editing & Deleting Rows

Each saved row shows a ✓ Save and × Delete button. Edit any field inline, then click ✓ to commit. A payment mismatch warning fires if the new amounts do not match the cost.

4.5   Submitting a Collection

When all boxes are entered, click Submit Collection. The collection moves to History with status Completed. A fresh Active tab is ready for the next pickup.

⚠ Submitted collections cannot be edited from the Active tab. Use History → Edit Entries to make corrections after submission.
05 History

The History tab lists every completed collection as a collapsible card showing date, branch, agent, total value, and box count.

5.1   Viewing Completed Collections

Click any card header to expand it and see the full box table. Click again to collapse.

5.2   Sorting

Click any column header inside an expanded card to sort by that column. Click again to reverse direction. Sort indicators (▲ ▼) show the active column and direction. Available columns: BOL Number, Customer, Destination, Size, Cost, Payment, Net.

5.3   Editing Entries (Audit Mode)

Each card has an ✏ Edit Entries button. Clicking it switches that card's rows into fully editable mode — the same inline inputs as the Active tab. A purple outline marks the card as being edited.

  • Fix check numbers, amounts, customer names, sizes, or destinations.
  • Click ✓ on each row to save that row's changes.
  • Click ✓ Done Editing (green button) to return to read-only view.
💡 Use Edit Entries to reconcile the web app against a physical ODS spreadsheet when discrepancies are found — e.g. a wrong check amount or a missing entry.
5.4   Print & PDF Export

Each card offers two export options in its action bar:

  • 🖨 Print Report: opens a print-formatted collection report in a new window.
  • 💾 Save PDF: generates and downloads a PDF of the collection report directly.
5.5   Re-opening a Collection

Click ↩ Re-open to move a completed collection back to the Active tab for continued editing. Useful if a collection was submitted prematurely or needs additional entries.

06 Reports

The Reports tab provides date-filtered tabular summaries. Set a date range at the top and choose a report type:

ReportShows

By Box SizeVolume and revenue broken down by size type
DetailedEvery collection with per-box line items

All report tables are sortable — click any column header to sort ascending; click again to sort descending. Reports can be printed or saved as PDF using the buttons at the top of the Reports view.

07 Analytics

The Analytics tab appears after at least two completed collections are recorded. It provides:

  • Period-over-period comparisons (current range vs prior range)
  • Total value, box count, average per collection, cash vs check split
  • Trend charts over time
  • Date range filters to focus on any window of history

Use the From / To date pickers at the top to set the primary period. A comparison period can optionally be set to benchmark against a prior span.

08 Search

The Search tab lets you find any box entry across all collections. Filter by one or more fields simultaneously:

  • Customer name: partial match on last or first name
  • BOL number: partial numeric match
  • Check number: exact or partial match
  • Date range: from/to collection date

Results show the collection date, BOL, customer, destination, size, and payment. Results update live as you type.

09 Contacts

The Contacts tab is a customer address book. Each contact can store name, phone numbers, email, two addresses (local and Philippines), and free-text notes.

Contacts are linked to shipment history — open a contact card to see every collection entry matched to that customer by name.

Import / Export

Contacts can be exported to a .csv file and imported back — useful for backup or moving between machines. Import supports both Overwrite (replace all) and Merge (add new, update existing by name) modes. Access import/export under Settings → Contacts.

10 Boxes Inventory

The 📦 Boxes tab tracks individual box units (distinct from collection entries). Each box record has a size, status, and an outgoing log.

  • Add boxes by size to your inventory
  • Mark boxes as available, in use, or shipped
  • Log outgoing box movements with date, quantity, and recipient notes
  • Print an inventory report from this tab

Set your branch number, branch title, and default agent name. These values pre-fill on new collections and appear on printed reports.

11.2   Pricing

Set the standard rate for each box size. Rates auto-fill the Cost field when a size is selected during entry.

11.3   Data File Management
  • Connect to Existing File: opens an Open dialog to link an existing .json data file. All data from the file loads into the app. Never overwrites on connect — read only.
  • Create New Data File: opens a Save As dialog to create a new empty file. Current app data is written to the new file immediately.
11.4   Backup & Restore

Backup downloads the entire dataset as a timestamped .json snapshot — independent of the live data file. Take backups before major imports or changes.

Restore merges a backup .json into the current dataset. Collections are merged by date, contacts by name. It does not wipe existing data.

11.5   Import from ODS Spreadsheet

See Section 13 for a full reference. In brief: click Import from Spreadsheet, select an .ods file, review the preview table (duplicates are auto-unchecked), optionally adjust pricing in the Pricing Override panel, and click Import Selected.

11.6   Check for Updates

Under the App section, click Check for Updates to compare your current version against the latest release on GitHub.

  • Up to date: shows your version and confirms it is current.
  • Update available: shows the new version number and a Download & replace file button.

Clicking Download opens a Save As dialog. Save over your existing LBC-CRM.html file to apply the update. Reopen the file in the browser to run the new version. Your data file is unaffected.

⚠ An internet connection is required to check for updates.
12 Multi-PC Shared Data Workflow

The app supports a shared JSON file workflow where multiple computers read from and write to the same data file on a network drive or synced folder.

Setup
  • Computer A creates the data file in a shared/synced location (e.g. a OneDrive folder visible to both machines).
  • Computer B opens the app and uses Connect to Existing File to link the same file.
  • Both computers now share the same dataset and save to the same file.
Staying in Sync
  • Auto-refresh on tab focus: whenever you switch back to the browser tab, the app re-reads the file and reloads data automatically.
  • Manual refresh: click the ↺ button in the header at any time to pull the latest data from the file immediately.
Important Considerations
  • One writer at a time: the app does not have real-time conflict resolution. Avoid two users saving simultaneously to prevent one overwriting the other's changes.
  • Cloud sync folders (OneDrive, Google Drive, Dropbox): work well for non-simultaneous use. Allow sync to complete before the second user opens the app.
  • Network drives: the drive must be mounted and accessible. If the drive is offline, the app falls back to browser local storage until reconnected.
💡 Click the Refresh button before starting a new collection entry to ensure you have the latest data from other users.
13 ODS Import Reference

The ODS import reads legacy .ods spreadsheet files (OpenDocument Spreadsheet format, as produced by the original LBC collection spreadsheet). It runs entirely in the browser — no server or external tool required.

13.1   Field Mapping Rules
ODS ContentImported AsRule
Row number column(skipped)Rows without a valid row number are ignored
BOL columnbolNumberMust be 8 or more digits — shorter numbers are treated as check numbers
Name columncustomerNameImported as-is
Size columnsizeSee size codes below
$$ in payment colpayment = cashDouble dollar sign marks a cash payment
1–5 digit numberpayment = checkShort numbers are check numbers, never BOL numbers
checkNum / $amtcheckTotalsParsed from the "Florida Agents Use Only" section; / is a literal separator, never division
Size Code Mapping
ODS ValueImported Size
L  or  J  (legacy pre-2025)Large (L)
Half, ½, 1/2Half Box
Jumbo (full word)Jumbo
Jr…Junior Box (Jr)
Irr + number 1–7Irregular (Irr1–Irr7)
⚠ The legacy size code J (used before approximately 2025) maps to Large. "Jumbo" requires the full word and is never confused with a single J.

Leading zeros on check numbers: ODS spreadsheets store numbers without leading zeros (e.g. 0096 becomes 96 in the cell value). The importer normalizes both the check totals and box row values by stripping leading zeros before matching, so check 0096 / $180 is correctly matched to its corresponding box row.

13.2   Pricing Override

By default, cash box costs are looked up from the current pricing table in Settings. For historical imports where rates were different, the import preview provides a Pricing Override panel (collapsed by default).

  • Expand the panel by clicking the ▶ arrow in the preview dialog.
  • Adjust rates for Large, Half, Jumbo, Jr, and Irregular.
  • These rates apply only to the current import run — they do not change your saved pricing settings.
  • Check payments are unaffected — their amounts always come from the spreadsheet totals, not from the pricing table.
💡 Before importing an old spreadsheet, check what rates were in effect at that time and enter them in the Pricing Override panel.
13.3   Duplicate Detection

The preview compares each collection's date against dates already in the app. Collections with a matching date are:

  • Shown grayed out in the preview table
  • Automatically unchecked (will not be imported by default)
  • Labeled with a ⚠ "exists" warning

You can manually check a duplicate to re-import it, which will add a second copy for that date — it does not overwrite the existing one. Delete the duplicate from History afterwards if it was unintentional.

⚠ Duplicate detection is by date only. Two legitimate collections on the same date will both show the warning — manually check the second one to import it.
